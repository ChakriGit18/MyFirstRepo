Welcome to the Git & GitHub Demo!

This page demonstrates how version control and collaboration work with Git and GitHub.

Steps to Use Git & GitHub

  1. Step 1: Initialize a Git repository in your project folder using git init.
  2. Step 2: Add your files to the staging area using git add ..
  3. Step 3: Commit your changes using git commit -m "Initial commit".
  4. Step 4: Push your changes to GitHub using git push origin main.

Collaborating on GitHub

To collaborate on GitHub:

  1. Fork a repository to create your own copy.
  2. Clone the repository to your local machine using git clone repository-url.
  3. Make changes to your files and push them to your GitHub fork.
  4. Create a pull request to propose your changes to the original repository.

Git Commands

Here are some important Git commands:

  • git status - Check the status of your working directory.
  • git log - View the commit history.
  • git pull - Fetch the latest changes from a remote repository.
  • git branch - List, create, or delete branches.
